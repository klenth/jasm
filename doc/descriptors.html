    Type descriptors

    JVM instructions that refer to fields and methods use descriptors that describe their types. This allows for overloading. The Java language allows overloading only of methods and only by parameter types, but JVM design permits overloading both fields and methods, including (for methods) by return type.

    Type descriptors

    Type (or field) descriptors describe a single type, such as the type of a field.

    Descriptor Meaning
    B byte
    S short
    I int
    J long
    F float
    D double
    C char
    Z boolean
    V void (only for method return types)
    Lclassname; Class object
    (classname must be given in internal form, using / instead of . to separate packages)
    [T Array of T, where T is another type descriptor
    (more than one [ means multidimensional array)
    Examples
    D                       # double
Ljava/lang/String;      # String
[I                      # int[]
[[Ljava/lang/Object;    # Object[][]
    Method descriptors

    Method descriptors describe the types (but not name) of a method’s parameters and return value.

    Syntax
    (params)return

    where params consists of zero or more type descriptors juxtaposed (no spaces or commas between them) — the types of the parameters — and return is a type descriptor for the method’s return type.

    Examples
    # Descriptors do not include names; the examples below use
# "f" as a placeholder name

# void f()
()V

# boolean f(int)
(I)Z

# String f(int, long, byte)
(IJB)Ljava/lang/String;

# String f(Object, String)
(Ljava/lang/Object;Ljava/lang/String;)Ljava/lang/String;

# void f(String[])
([Ljava/lang/String;)V
